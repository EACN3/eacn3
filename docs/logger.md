# 日志系统 Logger

## 定位

**共享工具包（Middleware）**——不是独立服务，而是通过注解（`@log_event`）嵌入各模块的日志记录库。任何涉及任务状态变更的函数挂载此注解即可自动记录。服务端和网络端各自实例化，各自记录本侧事件。

## 职责

记录任务全生命周期的状态变更、通信记录与执行过程。

> Agent 之间的直接通信（A2A 会话）不经过平台，Logger 无法记录其内容。

---

## 记录内容

- 任务全生命周期的状态变更
- 经过平台的通信记录（服务端记录本地通信，网络端记录任务流转通信）
- 每个 Agent 当前的执行状态

---

## 注解机制

日志系统提供注解函数 `@log_event`，所有涉及任务状态变更的函数必须挂载此注解。注解自动在函数执行前后拦截，记录上下文并更新日志，无需在函数内部手动调用。

```python
# 示例：任务状态变更时自动记录
@log_event
def submit_result(task_id, agent_id, content):
    ...

# 示例：竞标时自动记录
@log_event
def submit_bid(task_id, agent_id, confidence, price):
    ...
```

注解记录的内容：

```
LogEntry
├── fn_name     ← 被调用的函数名
├── args        ← 函数入参（含 from_id、to_id、content 等）
├── result      ← 函数返回值
├── timestamp
└── error       ← 若函数抛出异常则记录
```

---

## 设计原则

- **持久化**：所有记录写入事件表
- **旁路，不阻塞**：记录失败不影响原函数执行
- **只写不改**：追加写入，不修改历史
- **运营者专属**：Agent 无法访问日志数据
