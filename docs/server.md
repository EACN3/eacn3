# 服务端 Server

## 定位

客户部署的平台基础设施实例，是 EACN 网络中的**长期存活对象**。每个服务端有独立身份，在网络端注册后持续存在，承载该客户下所有 Agent 和工具的注册、匹配、声誉计算与日志记录。

---

## 身份

```
ServerCard
├── server_id            ← 全局唯一标识，注册时由网络端分配
├── version              ← 服务端软件版本号
├── endpoint             ← 服务端对外通信地址
├── owner                ← 所属客户标识
├── registered_at        ← 首次注册时间
└── status               ← 在线 | 离线
```

---

## 生命周期

```
注册
  └─→ 在线（正常运行，Agent 可注册，事件可上报）
        ├─→ 离线（心跳超时，网络端标记；恢复心跳后自动回到在线）
        └─→ 注销（客户主动注销，网络端清除该服务端及其下所有 Agent 的发现索引）
```

---

## 注册与认证

```
服务端首次启动
  └─→ 向网络端提交 ServerCard（version + endpoint + owner）
        └─→ 网络端校验
              ├─→ 版本低于最低支持版本 → 拒绝，提示升级
              └─→ 通过 → 分配 server_id，纳入已知服务端列表
```

后续每次通信，服务端携带 `server_id` 做身份识别。版本协商在注册时完成，后续大版本更新时再补充独立的 Gateway 协议层。

---

## 心跳与状态

服务端定期向网络端发送心跳，网络端据此维护状态：

- **在线**：心跳正常，该服务端下的 Agent 参与全局发现和匹配
- **离线**：心跳超时，网络端将该服务端下所有 Agent 从发现索引中临时摘除；心跳恢复后自动恢复
- 心跳只做存活检测，不携带业务数据

---

## 声誉

服务端自身有声誉，与 Agent 声誉独立：

- **用途**：加权该服务端上报的声誉事件。高声誉服务端上报的事件权重高，低声誉或新服务端权重极低
- **计算**：由网络端根据该服务端下 Agent 的跨服务端协作表现聚合得出
- **冷启动**：新注册服务端声誉为零，声誉事件在初期窗口内不计入全局，需通过跨服务端协作逐步建立

服务端声誉不是独立计算的新模块，而是 Reputation（网络端）的一部分——`receive_event` 时按 `server_id` 加权。

---

## 网络端的服务端注册表

网络端维护所有已注册服务端的 ServerCard，提供 `server_id → endpoint` 的查找能力。这是推送投递链的最后一环：

```
三层数据各自独立：
  DHT:        domain    → {agent_id}     （谁能干这活）
  AgentCard:  agent_id  → server_id      （agent 属于谁）
  ServerCard: server_id → endpoint       （server 在哪）
```

服务端注册表不参与发现逻辑（那是 DHT 的事），只负责：
- 存储 ServerCard（server_id、endpoint、status）
- 推送时提供 endpoint 查找
- 心跳超时时更新 status

---

## 与其他模块的关系

| 模块 | 关系 |
|------|------|
| Registry | 服务端内的模块，管理该服务端下 Agent 的注册 |
| Reputation（网络端） | 按 server_id 加权该服务端上报的事件 |
| 版本协商 | 注册时完成版本校验，后续大版本更新时再补充独立的 Gateway 协议层 |
| Discovery | 服务端离线时，其下 Agent 从发现索引中摘除；DHT 只存 domain → agent_id，不存 server 信息 |
| Push（网络端） | 推送事件时，通过 agent_id → server_id → endpoint 链路投递到服务端 |
| Logger（服务端） | 服务端内的模块，记录本地事件并上报网络端 |

---

## 设计原则

- **身份持久**：server_id 一经分配不变，客户重启服务端后重连即可，无需重新注册
- **状态简单**：只有在线和离线两个状态，由心跳自动维护
- **声誉挂钩**：服务端声誉影响其下所有 Agent 的全局声誉事件权重，但不影响 Agent 本地声誉
- **注销级联**：服务端注销时，其下所有 Agent 从网络端发现索引中清除
